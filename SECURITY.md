# Безопасность

## Секреты

Не коммитьте в репозиторий:

- `.env`, `.dev.env` и любые файлы с токенами ботов, API-ключами Yandex, паролями 1С;
- выгрузки информационных баз и файлы публикации Apache с реальными путями и паролями (`default.vrd`).

Используйте `.env.example` как шаблон.

## Если секрет попал в Git

1. Немедленно **отзовите** скомпрометированный токен / ключ / пароль (BotFather, Yandex Cloud, пользователь 1С).
2. Удалите секрет из истории репозитория или создайте новый репозиторий без истории.
3. Замените значения в локальном `.env`.

## Сообщить об уязвимости

Опишите проблему через Issues репозитория (без публикации эксплойта и реальных секретов).
